        Status do Azure DevOps
        O que cada status significa e o que você deve fazer quando vê cada um.
        Como funciona o fluxo normal: Proposed → Active → Awaiting QA → Resolved → Closed.
        Se algo deu errado: vai para Revised (volta para corrigir) ou Refused (descartado). Paused = travado, exige atenção imediata.

        Testes & Pareceres
        Como saber se os testes estão sendo feitos e como cobrar pareceres registrados.
        Status de Teste — o que cada um significa
        StatusTraduçãoO que significa
        ✅ PassedAprovadoTestou e funcionou — está pronto para Resolved
        ❌ FailedReprovadoTestou e encontrou erro — deve ir para Revised
        🚫 BlockedBloqueadoNão conseguiu nem testar — há um impedimento
        ⏸️ Not RunNão executadoCaso de teste criado mas nunca executado
        🔵 ActiveAtivoCaso de teste criado, aguardando execução
        ⚠️ Armadilha mais comum
        Item Resolved ≠ Item testado
        Muitos devs marcam como Resolved sem que o QA tenha validado. Sempre cruze: item Resolved → tem Test Case com status Passed? Se não tiver, o item não foi testado formalmente.
        Como verificar se os testes estão acontecendo
        • 1️⃣ No DevOps, vá em Test Plans — lá ficam os casos de teste por projeto
        • 2️⃣ Verifique se o item em Awaiting QA tem um Test Case criado
        • 3️⃣ Veja se o Test Case foi executado (status Passed, Failed ou Blocked)
        • 4️⃣ Item com Awaiting QA e Test Case em "Not Run" há mais de 2 dias = teste não aconteceu
        • 5️⃣ Sempre pergunte: os testes foram executados em ambiente de homologação?
        Quando cobrar um Parecer
        • 📝 Item foi de Resolved para Revised → QA deve ter registrado o motivo no Discussion
        • 📝 Item está Paused → deve ter comentário explicando o bloqueio
        • 📝 Item foi Refused → motivo da recusa deve estar documentado
        • 📝 Teste Failed → descrição do erro deve estar no item
        Frase assertiva para cobrar:
        "Por favor, registre o motivo diretamente no item do DevOps, na aba Discussion. Assim eu consigo acompanhar sem precisar perguntar."
